先记住 4 句话

  • 这题本质上是在做 “更聪明的代码生成”，不是重新发明算子。
  • 你要做的是 识别某些可确定的输入形态，然后走更快的特化路径。
  • 任何特化都必须保留 fallback 通用路径，否则隐藏测试会炸。
  • 评审看四件事：正确、命中特化、生成代码更简洁、运行更快。

使用方式

主干主题：比赛目标、系统、交付、评测
中间概念：把黑话翻成人话
最小叶子：不能再拆的基础概念

读图顺序

  • 先点根节点，建立“这题到底要我干嘛”的整体感。
  • 再看“系统长什么样”，理解 `tensor`、`shape`、`stride`、`layout`。
  • 然后看“特化为什么会更快”，理解 `mask`、`pointer arithmetic`、`fast path`。
  • 最后看“如何交卷”，理解测试、benchmark、报告、评分公式。

新手建议

  • 第一次做这题时，最容易落地的是 `contiguous fast path` 或 `broadcast / scalar fast path`。
  • 别一上来追性能，先能稳定判断“什么时候命中特化，什么时候必须回退”。
  • 看生成代码时，不要只盯 runtime，也要数 `mask`、`stride` 和 pointer 表达式有没有减少。

节点说明

默认选中

T1-2-1 赛题全貌

这道题要求你在 NineToothed 的代码生成系统里增加 1 到 2 类“可靠可判定”的特化逻辑，让某些输入在生成 Triton 代码时更简洁、更少冗余表达式，并在不满足条件时安全回退到原来的通用路径。

你最后要交什么

代码：主要改 `generation.py`、`aot.py`，必要时补一点 `tensor.py` helper。
测试：命中特化、fallback 正确性、generated source 结构测试都要有。
benchmark：至少 2 个命中特化 case + 2 个 fallback case，并记录速度和代码指标。
报告：解释 baseline 哪里弱、你怎么特化、为什么安全、收益和盲区是什么。
